# Plaster and Rendering Solutions - Project Guide

## 🎯 Project Overview
This project is a premium, high-end "Apple-style" website for a Plaster and Rendering Solutions business. The design must be exceptional, avoiding typical templates, and prioritizing a sleek, modern, and highly dynamic user experience.

## ✨ Core Requirements & Aesthetics
- **Apple-Style Design**: Minimalist layout, generous whitespace, large and modern typography (e.g., SF Pro, Inter, or Outfit), and subtle glassmorphism effects.
- **High-End Animations**: Smooth micro-interactions, scroll-triggered reveal animations, and seamless page transitions (utilizing libraries like Framer Motion or GSAP). 
- **Consistent Theming**: Strict adherence to a tailored, premium color palette throughout all pages. Avoid default or basic colors; use curated, harmonious tones that reflect high-end craftsmanship (e.g., deep charcoal, soft plaster creams, warm earthy accents, and sleek dark modes).
- **Dynamic & Alive**: The interface must feel responsive to the user. Use precise hover effects, smooth gradients, and interactive elements.

## 📄 Required Pages
1. **Home Page**: High-impact, immersive landing page with breath-taking visuals and animations.
2. **Service Pages**: Detailed, visually rich breakdowns of individual plastering and rendering services.
3. **FAQ Page**: Interactive, smoothly animated accordion-style questions and answers.
4. **Policy Pages**: Clean, readable, and beautifully structured text (Privacy Policy, Terms of Service).
5. **Contact Page**: Elegant, highly functional contact form with premium input styling and real-time validation.

## 🛡️ Self-Checking & Resilience Architecture
The application must "check itself after any issues" to ensure maximum reliability:
- **Error Boundaries**: Implement robust error boundaries (e.g., React Error Boundaries) to catch UI crashes and display elegant, on-brand fallback screens instead of breaking the app.
- **Form Validation**: Real-time, rigorous client-side validation with smooth, non-intrusive error messages.
- **Robust State Management**: Ensure the website maintains stable state and can recover gracefully from network failures or API timeouts.
- **Automated Testing**: Write tests for critical flows to ensure the application self-verifies its integrity before and after deployments.

## 🛠️ Development Rules for AI Assistants
1. **No Basic Designs**: Never implement generic MVPs. Every component built must WOW the user.
2. **Componentized Architecture**: Build highly reusable, cleanly separated UI components for consistency.
3. **Responsive Excellence**: All designs must be flawlessly responsive across mobile, tablet, and desktop natively.
4. **SEO Best Practices**: Every page must implement strict SEO rules (semantic HTML5, meta tags, and proper `<h1>` to `<h6>` hierarchy).
